How Decoding Works

For any grayscale pixel:

value ∈ 0..255

the glyph is the nearest anchor value.

Example:

value = 150
nearest = 151
glyph = —

This makes the decoder robust to noise in the sandpile evolution.

Glyph Index

Internally we can assign the 12 symbols indices:

0  @
1  N
2  \
3  `
4  q
5  „
6  …
7  —
8  ¦
9  ©
10 ¹
11 É

So decoding becomes:

anchors = [64,78,92,96,113,132,133,151,166,169,185,201]
glyphs  = ['@','N','\\','`','q','„','…','—','¦','©','¹','É']

def decode(value):
    idx = min(range(len(anchors)),
              key=lambda i: abs(value-anchors[i]))
    return glyphs[idx]
Why This Works Well With Sandpiles

Sandpile states tend to form plateaus and sharp boundaries.
Using anchor luminances means:

small fluctuations stay mapped to the same glyph
boundaries produce glyph transitions
the visual rainbow ordering stays coherent

So the glyph stream becomes stable even under dynamic evolution.

# Supersingularity Invariants



## Semantic invariants

- Graph semantics dominate image semantics.
- Python-directed semantic control currently dominates compiled acceleration helpers.
- `scope singularity init` must remain a deterministic scoped constructor and must not silently become an arbitrary mutation command.
- Local state uniqueness is provisional; stable relation classes are more important than naive one-off distinctness.
- A sink is part of the sandpile model, not optional metadata.
- Reduced Laplacian conventions must remain explicit.
- Edge multiplicity and degree must not be discarded.
- Stabilization is a semantic event, not merely a visual effect.
- The production pointer is the current scope boundary for safe continuation.
- Scope expansion into wider `Codeletteria` context is allowed only when it still aligns with the active production pointer and declared theory anchors.

---

## Interval and surface invariants

- Interval partitions are part of the semantics, not a rendering trick.
- Every cell of a valid surface must belong to exactly one declared interval family (no “free” values).
- Interval representatives (e.g. the 12-glyph rainbow) are canonical anchors for encoding and decoding, not arbitrary stylistic choices.
- Local computations (e.g. expressions like `i + j^2`) must respect the active interval-dependency graph:
  - Given `i ∈ I_a` and `j ∈ I_b`, the result must land in the required `I_c` when the rule is in force.
- Surface validity is determined by interior behavior:
  - **Interior-valid:** all interior points satisfy the relevant interval-dependency rules for the current scope.
  - **Mixed interior:** some interior points satisfy, some violate; this is transitional and must not be silently treated as stable.
  - **All-interior-invalid:** every interior point violates a given dependency rule; this is a semantic classification event, not just an error.
- For certain relation classes or scopes, “all-interior-invalid” is an iff characterization:
  - A surface belongs to class `K` **iff** every interior point fails rule `R` under the active interval-dependency graph.
- Interval semantics must remain distinguishable from raw greyscale/image semantics in code, manifests, and tests.

---

## Five-instruction semantic core

These invariants assume that all higher-level behavior can be expressed in terms of five conceptual instructions:

- `scope`  
  - Selects an interval family / relation class / address family.
  - Must not implicitly widen beyond the active production pointer without an explicit scope transition.

- `singularity`  
  - Chooses or detects a distinguished point (seed, fixed point, ideal expression).
  - Must be derivable from recorded context, not an ad hoc override.

- `init`  
  - Materializes a surface from a seed under the current scope.
  - Must remain a deterministic scoped constructor and must not silently become an arbitrary mutation command.

- `goto`  
  - Traverses along a successor edge (filesystem step, isogeny edge, Collatz move, sandpile successor, etc.).
  - Must respect traversal compatibility: it either preserves declared invariants or intentionally transitions between classes in a documented way.

- `iff`  
  - Branches on global invariants (e.g. interior-valid vs all-interior-invalid, class membership via interval rules).
  - Must not be reduced to a generic boolean without preserving which invariant is being tested.

**Semantic closure invariant**

- Any valid program in this universe should be representable as compositions of:

  `scope → singularity → init → (goto / iff)*`

  such that all intermediate surfaces satisfy the applicable semantic, interval, and traversal invariants for their scopes.
