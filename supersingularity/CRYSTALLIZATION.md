Supersingularity Crystallization

Purpose

- This file records the current seed state as a reference shape for future building.
- It is not the final architecture. It is the current stable center.

Current seed

- Python remains the semantic control layer.
- Compiled helpers remain subordinate acceleration seams.
- The viewer is a terminal-canvas projection of machine state, not the semantic authority.
- Startup is generated procedurally from an initialization vector.
- The startup recursion has a global horizon and a smaller materialized prefix.
- Stored `.pgm` artifacts remain state surfaces and storage substrates, not the primary mathematics.

What is stable right now

- `graph.py` is the graph-semantic layer.
- `dag.py` is the typed derivation and event layer.
- `simulation.py` is the recursive state generation and storage layer.
- `viewer.py` is the ASCII projection layer.
- `ternlsb.py` is the hidden instruction/carrier layer.
- `compiled.py` is a Python-controlled bridge to compiled helpers.

Current machine model

- There is a distinction between:
  - global recursive horizon
  - materialized recursive prefix
  - persisted checkpoints
  - current machine state
- Startup should be understood as generated computation, not a theatrical animation.
- The machine should remain at the state it reaches.
- Apparent non-uniqueness in local sandpile play should be allowed to crystallize into higher-order uniqueness through recurrence, collapse, and ordered relation.
- The idealized universal self-distinguishing argument should be treated as emerging from Oxfoi invariance plus supersingular/isogeny ordering plus sandpile recurrence, not from a flat diagonal metaphor alone.

Viewer rule

- The viewer should feel like one stable terminal surface.
- It should not feel like a slideshow, menu, or debugger unless explicitly used that way.
- Terminal output should present state, not frame logic.

Production-pointer rule

- The production pointer remains the current safe boundary for continuation.
- Outer scope expansion into wider `Codeletteria/` is valid only when it still aligns with that boundary.

Crystallization targets

- Make invariants increasingly explicit.
- Make startup/configuration semantics explicit.
- Reduce accidental behavior.
- Convert recurring concepts into typed objects.
- Keep the seed reference intelligible while allowing growth.

What must not drift

- Python must not lose semantic authority to compiled helpers.
- The graph layer must not be collapsed into image-only thinking.
- The viewer must not replace the machine model with a merely decorative UI.
- Stored state must remain parseable and reproducible.
- Later work must refine this seed rather than silently replace it.

Near-term building directions

- Formalize startup and initialization vectors further.
- Promote important state features into typed semantic records.
- Keep improving the viewer as a stable machine-state surface.
- Connect stego/program semantics more deeply into lattice and graph structure.
- Clarify what counts as progress in the DAG, hypergraph, and production-pointer layers.
